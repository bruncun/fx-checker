# Architecture Decision Records

This directory records the early technical and product-shaping decisions for FX Checker.

The project uses lightweight ADRs because this is a small hackathon-style application. Each record should capture the decision, why it was made, and the trade-offs we accepted. ADRs are not meant to freeze the codebase; they are a memory aid for future contributors.
